* ledger设置循环交易

** ledger-cli如何支持循环交易

#+begin_src markdown
在 **ledger-cli** 里，“循环交易” 一般是指一种定期重复发生的交易，比如每周/每月的工资、房租、水电费、转账等。
Ledger 本身并不会像记账软件那样自动生成未来的交易，但它有两种机制可以支持这种场景：

---

## **1. 用 `~` 定义周期性交易（Periodic Transactions）**

Ledger 的 journal 文件可以直接写定期规则，让它在报表时自动扩展出这些交易。
语法大致是：

```ledger
~ every month on the 5th
    Expenses:Rent               2000 CNY
    Assets:Bank:ICBC
```

### 常用的周期表达式

,* `~ monthly` / `~ every month` → 每月
,* `~ weekly` / `~ every week` → 每周
,* `~ daily` / `~ every day` → 每天
,* 可以指定日期：`~ every month on the 15th`
,* 可以指定范围：`~ weekly from 2025/01/01 until 2025/12/31`

例如：

```ledger
~ every week
    Expenses:运营商:中国联通    49.75 CNY
    Assets:Bank:工商银行
```

这个不会直接写入账本，只会在你运行报表命令时自动计算这些交易的总额。

---

## **2. 生成实际交易（Materializing Periodic Transactions）**

如果你想让循环交易“落地”成普通交易（方便将来修改或导出），可以用 Ledger 自带的扩展功能：

```bash
ledger -f ledger.dat print --period "from 2025/01/01 to 2025/12/31"
```

配合 `--actual` 可以生成真实交易而不是仅显示。

也可以用 **ledger-autoschedule**（第三方脚本）将周期交易展开为实际的交易条目存到文件里。

---

## **3. 注意事项**

,* 周期交易不会自动影响期初余额，因为它是计算时临时展开的。
,* 如果你的循环交易金额会变化（比如水费），那最好生成实际交易手动调整。
,* Ledger 的日期计算遵循 `boost::date_time` 规则，复杂的周期需要测试一下（例如“每月最后一天”要用 `on last day of month`）。

答：
#+begin_src markdown
我看出问题了――你已经写了 `~ daily` 的周期交易，但运行 `ledger bal` 时它没显示出来，是因为：

1. **Ledger 默认不展开 periodic transaction**
   周期交易（`~ daily` 这种）只是一个“规则”，Ledger 在报表里只有在**查询区间有符合条件的日期**时才会临时生成虚拟交易。

2. **你的命令少了 `--period` 或指定时间范围的方式**
   `-b 08/13` 只是设置**起始日期**，但 Ledger 只会在该日期或之后符合周期规则的日期里计算。
   如果今天是 8/13，`~ daily` 会从 8/13 开始每天都加一笔，但 `bal` 只显示累计余额，不会列出每一天的交易。

3. **Periodic transaction 默认不影响总余额（unless `--forecast`）**
   如果你想在余额中看到它的影响，需要加 `--forecast` 选项。

---

### 解决方法

#### 如果只是想让它在余额里生效

```bash
ledger -f ledger.ledger bal -b 2025/08/01 --forecast
```

`--forecast` 会把周期规则展开成未来的交易（直到你设定的结束日期）。

#### 如果想查看具体展开的交易

```bash
ledger -f ledger.ledger register --period "2025/08/01 to 2025/08/31" --forecast
```

#### 如果要生成实际交易到文件

```bash
ledger -f ledger.ledger print --forecast --period "2025/08/01 to 2025/08/31" > expanded.ledger
```

这样 `expanded.ledger` 里就是每天的那笔 49.75 CNY，跟普通交易一样。

,* `--forecast` 是让规则“生效”的关键。
,* 如果要真正存成普通交易，需要用 `print` 导出。

答：
#+begin_src markdown
你这个报错是因为 **`--forecast` 在新版 ledger-cli 已经被改成 `--forecast-while`**，而且它不是单独开关，而是需要跟一个“条件表达式”。

也就是说，旧版本的：

```bash
ledger ... --forecast
```

在你这个版本里变成了：

```bash
ledger ... --forecast-while <EXPRESSION>
```

---

## 用法示例

比如你想让它一直生成交易直到 2025/08/31：

```bash
ledger -f ledger.ledger bal \
    -b 2025/08/01 \
    --forecast-while "date <= [2025-08-31]"
```

这样 ledger 会展开 `~ daily` 规则，从 8/1 到 8/31 每天一笔。

如果想看具体的交易明细而不是余额：

```bash
ledger -f ledger.ledger register \
    --forecast-while "date <= [2025-08-31]"
```

---

## 解释

,* `--forecast-while` 会不断展开 periodic transactions，直到条件不成立。
,* 条件里可以用 `date`（交易日期）、`amount` 等 ledger 表达式。
,* 日期要用方括号 `[YYYY-MM-DD]`，这样它会被当成日期而不是字符串。
